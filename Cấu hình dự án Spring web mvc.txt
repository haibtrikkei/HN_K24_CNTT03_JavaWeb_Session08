


Validate:
	1. Dữ liệu phải nhập (tất cả các trường)
		radio, checkbox, select, date: Chọn
	
	2. Các kiểu dữ liệu đúng định dạng:
		- Kiểu email
		- Kiểu số điện thoại
		- Kiểu mã (mã sản phẩm, mã sinh viên)
		- Kiểu số: Phải trong phạm vi (ví dụ: từ 10 - 30, tối thiểu 0)
		
	3. Các dữ liệu phải duy nhất
		- Email, số điện thoại, tên danh mục, ...
		
* Đa phần sẽ validate bên client:
	- Dữ liệu phải nhập
	- Đúng định dạng
* Các trường hợp validate bên server:
	- Dữ liệu phải duy nhất


@NotNull	: Trừ String
@NotEmpty	: Không rỗng (nhận dấu cách - khoảng trắng)
@NotBlank	: Không rỗng, không khoảng trắng
@Min
@Max
@Size
@Pattern	: Kiểu String

Bước 3: Cấu hình ứng dụng
	- WebConfig : implements WebMvcConfigurer
		Cấu hình view thymeleaf:
		  + SpringResourceTemplateResolver
		  + SpringTemplateEngine
		  + ThymeleafViewResolver
		  
		Cấu hình Upload file:
		  + Cấu hình giới hạn kích thước file upload trong WebInit
			Khai báo hằng số:
				private final String TMP_FILE = "";
				private final Long MAX_FILE_SIZE = 10*1024*1024L;
				private final Long MAX_REQUEST_SIZE = 50*1024*1024L;
				private final Integer FILE_SIZE_THRE_SHOLD = 10*1024*1024;
			Override method:
				@Override
				protected void customizeRegistration(ServletRegistration.Dynamic registration) {
					registration.setMultipartConfig(new MultipartConfigElement(TMP_FILE, MAX_FILE_SIZE, MAX_REQUEST_SIZE, FILE_SIZE_THRE_SHOLD));
				}
		  + Cấu hình MultilpartResolver, và resourceHandler trong WebConfig
			
	- WebInit	: extends AbstractAnnotaionConfigDispatcherInitializer

* Thực hiện validate form:
1. Tạo lớp DTO chứa dữ liệu form truyền lên controller

3. Tạo class controller
4. Tạo view insert
